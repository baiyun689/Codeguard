# Trace 报告时间戳设计

**日期**：2026-07-09
**状态**：已确认

## 目标

让每份 Trace HTML 在文件名和页面头部都明确显示生成时间，便于区分多次审查产物。

## 设计

- 继续使用 `TraceReport.timestamp` 作为唯一时间来源。
- 文件名格式改为 `trace-YYYYMMDD-HHMMSS-<run-id前8位>.html`。
- 页面头部增加“生成时间”指标，显示 `TraceReport.timestamp` 原值。
- 时间使用生成报告进程的本地时区。
- 非法或缺失 timestamp 时，文件名使用当前本地时间兜底；页面仍显示原始值或 `unknown`。
- 不改变 Trace 数据结构、事件内容和输出目录规则。

## 验收

- 固定 timestamp `2026-07-09T20:30:45` 生成文件名 `trace-20260709-203045-abc12345.html`。
- HTML 页面包含生成时间 `2026-07-09T20:30:45`。
- 现有报告 JSON 和安全嵌入行为保持不变。
