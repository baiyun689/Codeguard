## 文件与路径 I/O（外部输入是否越出受控目录、上传可执行内容或解压逃逸）

### 典型模式
```java
Path target = Paths.get(uploadRoot).resolve(file.getOriginalFilename());
Files.copy(file.getInputStream(), target);
```
`resolve` 后未 `normalize` 并校验 `startsWith(base)`；直接使用请求参数、Zip entry 名称或绝对路径读取/写入；`MultipartFile.transferTo` 不校验文件类型、大小、存储目录；解压循环不防 `../` 和压缩炸弹。

### 判定要点
确认路径的可控来源、规范化后的边界、符号链接与覆盖行为；上传类型要依据内容/白名单而非仅扩展名。对于下载，确认 base 目录是否过宽以及对象授权是否已验证。检查解压累计大小、条目数和每条目标路径。

### 严重级别参考
CRITICAL：攻击者能读写任意服务器文件、写入可执行 web root 或 zip slip 成功。WARNING：可控路径可能越出专用目录、覆盖受限文件，或上传/解压边界存在可利用缺口。INFO：只用于已证明外部输入影响路径、但读写范围严格受限的具体弱点；仅建议统一 helper 不报告。

### 已知误报判例
文件名由服务端 UUID 生成；`target.normalize().startsWith(root)` 且 root 为专用不可执行目录；对象存储 SDK 不接受本地路径；下载前服务层已按资源所有者授权。

### 排除项
文件句柄泄漏属于 RESOURCE_LIFECYCLE；路径来自可信配置但配置含秘密属于 CONFIG_SECURITY；SQL 或 shell 语法拼接属于 INJECTION。
