## 性能行为（改动是否造成可观测的延迟、吞吐或容量退化）

### 典型模式
```java
for (Long id : ids) {
    results.add(client.fetch(id));
}
```
请求路径循环查询数据库/远端服务，读全表后内存分页，在 Web 线程做大文件/加密/阻塞等待，缓存失效导致高基数 key 或同步热点重建。

### 判定要点
确认循环规模、调用是否可批量化、索引和分页是否匹配；区分低频后台任务与高 QPS 主路径。检查超时、线程池和背压，要求证据表明改动位于实际热点而非只凭“循环”判断。

### 严重级别参考
CRITICAL：高流量主链路可耗尽连接/线程并引发级联不可用。WARNING：明显 N+1、无界加载或同步阻塞会造成超时。INFO：微优化、低频任务或尚无影响证据的建议。

### 已知误报判例
集合有严格小上限；repository 用批量查询和二级缓存；任务离线运行且有资源配额；异步 client 的实际调用不阻塞请求线程。

### 排除项
资源未释放属于 RESOURCE_LIFECYCLE；数据库更新语义错误属于 SQL_DATA_ACCESS；复杂分支可读性属于 COMPLEXITY_CONTROL_FLOW。
